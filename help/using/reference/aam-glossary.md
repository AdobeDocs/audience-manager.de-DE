# Glossar{#glossary}

Definitionen und Links zur weiteren Lesbarkeit.

## A-B {#a-b}

**Algorithmische Modellierung**

Verwenden Sie diese Option, um [!UICONTROL Algorithmic Modeling] die Reichweite über das Haupt der Benutzer hinaus zu erweitern, die Sie identifiziert haben. Mit dieser Funktion können Sie neue, einzigartige Zielgruppen durch automatisierte Datenanalyse entdecken. Verwalten Sie Ihre [!UICONTROL Algorithmic Models] in **[!UICONTROL Audience Data > Models]**.

Siehe [Grundlagen zu Algorithmischen Modellen](../features/algorithmic-models/understanding-models.md#understanding-models).

<br> 

**BAAAM**

[!UICONTROL Bulk Management Tools]. Die [!UICONTROL Bulk Management Tools] in [!DNL Audience Manager] sind ein Microsoft Excel-basierter Werkzeugsatz, mit dem Sie mehrere Objekte mit einem einzigen Vorgang erstellen, ändern oder löschen können. Sie können mit Datenquellen, abgeleiteten Signalen, Zielen, Ordnern, Segmenten und Eigenschaften arbeiten. Die Funktion verwendet eine Microsoft Excel-Tabelle mit Makros, die sichere, authentifizierte Aufrufe an [!DNL Audience Manager] die apis vornehmen.

Siehe [Massenverwaltungswerkzeuge](../reference/bulk-management-tools/bulk-management-intro.md).

## C-D {#c-d}

**CDF**

[!UICONTROL Customer Data Feed]. Eine [!UICONTROL CDF] Datei stellt einen Massen-Download von Daten dar, die erfasst werden, [!DNL Audience Manager] und ermöglicht Ihnen die Arbeit [!DNL Audience Manager] mit Daten außerhalb der von unserer Benutzeroberfläche festgelegten Beschränkungen. Eine [!UICONTROL CDF] Datei enthält dieselben Daten, die ein [!DNL Audience Manager] Ereignisaufruf an `/event`unsere Server sendet. Dazu gehören Daten wie Benutzer-IDs, Eigenschaften-IDs, Segment-IDs und alle anderen Parameter, die durch einen Ereignisaufruf erfasst werden.

Siehe [Kundendatenfeeds](../features/cdf-files.md).

<br> 

**CRM-ID**

Die CRM-ID ist die ID, mit der Kunden Benutzer in ihrem eigenen CRM-System identifizieren. Anstelle der CRM-ID verwenden wir den Begriff DPUUID in Audience Manager.

Siehe DPUUID im [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Kundenadressierbare Zielgruppe**

In [Addressable Audience](/help/using/features/addressable-audiences.md)stellt diese Metrik Geräte dar, die:
* Entweder eine regelbasierte oder eine aufgenommene Eigenschaft während des Look-Back-Fensters
   **UND**
* Eine ID wird unabhängig vom Synchronisierungszeitpunkt mit dem ausgewählten Ziel synchronisiert.

<br> 

**Kundenattribute**

Siehe [Kundenattribute](https://marketing.adobe.com/resources/help/en_US/mcloud/attributes.html) in der [!DNL Experience Cloud Core Services] Produktdokumentation.

<br> 

**Kundenübereinstimmungsrate**

Customer Addressable Audience ÷ Customer Total Audience Express as a %. Siehe [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**Zielgruppe für Kunden insgesamt**

In [Addressable Audience](/help/using/features/addressable-audiences.md)stellt diese Metrik eine Anzahl von Geräten dar, die entweder eine regelbasierte Eigenschaft in Ihren Eigenschaften oder eine Onboardanmerkung aus Ihren Offline-Dateien während des Look-Back-Fensters haben.

<br> 

**demdex.net**

Demdex. net ist eine Legacy-Domäne, die gesteuert [!DNL Adobe]wird. Es spiegelt [!DNL Audience Manager]den ursprünglichen Namen, den Vorakquise-Namen ( [!DNL Demdex]) wider. [!DNL Adobe] im [!DNL Demdex] Jahr 2011 erworben und das Unternehmen erneut [!DNL Audience Manager]markiert. Alle HTTP-Aufrufe an `demdex.net` Domänen sind Aufrufe an [!DNL Adobe].

Siehe [Aufrufe an die Domäne „demdex.net“](../reference/demdex-calls.md).

<br> 

**DAID**

[!UICONTROL Device Advertising IDs] sind eindeutige Gerätekennungen zur Identifizierung eines Mobilgeräts. Diese IDs werden vom Gerätehersteller und nicht von Adobe zugewiesen. Wir unterstützen ios- und Android-Geräte-IDs in [!DNL Audience Manager].

Siehe [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Ziel**

Ein [!DNL Audience Manager]Ziel ist ein beliebiges anderes System (Anzeigen-Server, DSP, Werbenetzwerk usw.) , für die Sie Daten freigeben möchten. In [!UICONTROL Destination Builder] unserer Benutzeroberfläche finden Sie die Tools, mit denen Sie diese Datenbereitstellungsprozesse erstellen und verwalten können. [!DNL Audience Manager] Zielfunktionen befinden **[!UICONTROL Audience Data > Destinations]** sich in.

<br> 

**DIL**

Die [!UICONTROL Data Integration Library] API-Bibliothek dient zur [!DNL Audience Manager] Erfassung von Benutzerinteraktionsdaten. Siehe DIL-API ( [Data Integration Library](../dil/dil-overview.md)).

<br> 

**dpm**

[!UICONTROL Data Provider Match]. Es weist interne [!DNL Adobe] Systeme darauf hin, dass ein Aufruf von [!DNL Audience Manager] oder der ID-Dienst Kundendaten zur Synchronisierung weiterleitet oder eine ID anfordert. Siehe [Aufrufe an die Domäne „demdex.net“](../reference/demdex-calls.md).

## E-F {#e-f}

**Experience Cloud ID (ECID)**

Zuvor benannte [!DNL Marketing Cloud] ID (MID oder MCID). Die [!DNL Experience Cloud] ID ist für den ID-Dienst von zentraler Bedeutung. Es handelt sich um einen eindeutigen und beständigen Bezeichner für Ihre Site-Besucher. Siehe Cookies und Experience [Cloud ID-Dienst](https://marketing.adobe.com/resources/help/en_US/mcvid/mcvid_cookies.html).

<br> 

**Ordnereigenschaften**

Automatische Gruppierung von Eigenschaften in Ihrer Ordnertaxonomie. Jeder Ordner in Ihrer Hierarchie erstellt automatisch eine Eigenschaft, die zur Definition von Segmenten verwendet werden kann.

Siehe [Ordnereigenschaften: Info](../features/traits/about-folder-traits.md).

<br> 

**Häufigkeitsklick**

Ein Limit, mit dem ein Advertiser einem Endbenutzer eine bestimmte kreative Darstellung anzeigen möchte. Sie können verschiedene häufige Capping-Ausdrücke in [!UICONTROL Segment Builder]konfigurieren.

Siehe [Neuigkeit und Häufigkeit](../features/segments/recency-and-frequency.md).

## G-H {#g-h}

**GAID**

Google Advertising ID, die eindeutige Geräte-ID, die Google auf Hardware-Geräten zuordnet, die das Android-Betriebssystem ausführen. Siehe [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**GUID**

Ein Akronym für Global eindeutige Kennung. Der Begriff GUID wird nicht [!DNL Audience Manager]verwendet. In unserem Fall ist die GUID die [!DNL Audience Manager] UUID.
Siehe [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

## I-J {#i-j}

**IDFA**

ID für Advertiser, die eindeutige Geräte-ID, die Apple seinen Produkten zuordnet. Siehe [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Inbound**

Der Prozess, mit dem Sie Zielgruppendaten aus anderen Quellen senden [!DNL Audience Manager]können. Siehe [Senden von Zielgruppendaten](/help/using/integration/sending-audience-data/send-audience-data.md).

<br> 

**Integrationscode**

Beim Arbeiten mit der [!DNL Audience Manager] Benutzeroberfläche oder API haben Sie die Möglichkeit, beim Erstellen von Eigenschaften, Segmenten oder Datenquellen einen Integrationscode hinzuzufügen. Integrationscodes dienen in diesen Fällen unterschiedlichen Zwecken:

* [!UICONTROL Traits]: Ein Integrationscode ist ein Feld für eine ID, SKU oder einen anderen Wert, der von Ihren internen Geschäftsprozessen verwendet wird. Optional.
* [!UICONTROL Segments]: Ein Integrationscode ist ein Feld für eine benutzerdefinierte ID oder andere unternehmensspezifische Informationen. Optional.
* [!UICONTROL Data Sources]: Integrationscodes sind erforderlich, wenn Sie geräteübergreifende Datenquellen erstellen, den Experience Cloud ID-Dienst verwenden oder mit [!UICONTROL Profile Merge Rules]arbeiten möchten. Weitere Informationen finden Sie unter [Datenquelle erstellen](../features/manage-datasources.md#create-data-source) .

## O-P {#o-p}

**PCS**

[!UICONTROL Profile Cache Server]. Die ist [!UICONTROL PCS] eine umfangreiche Datenbank, die auf Apache Cassandra ausgeführt wird. Speichert Daten, die für aktive Benutzer von Server-zu-Server-Übertragungen und vom Server [!UICONTROL DCS]empfangen wurden. [!UICONTROL PCS] Daten bestehen aus Geräte-IDs, authentifizierten Profil-IDs und den zugehörigen Eigenschaften.

Siehe [Datenerfassungskomponenten](../reference/system-components/components-data-collection.md).

<br> 

**Profillink**

Siehe Regeloptionen [für die Profilzusammenführung definiert](../features/profile-merge-rules/merge-rule-definitions.md).

<br> 

**Regeln für die Profilzusammenführung**

[!UICONTROL Profile Merge Rules] können Sie steuern, welche Datenart für die Segmentierung [!DNL Audience Manager] verwendet werden soll.

Siehe Regeloptionen [für die Profilzusammenführung definiert](../features/profile-merge-rules/merge-rule-definitions.md).

## Q-R {#q-r}

**Neuerstellung**

Die Aktion, mit der sich ein Besucher auf Ihrer Site für eine Eigenschaft qualifiziert. Mit dem Tool [&quot;Besucherprofil Viewer](../features/visitor-profile-viewer.md) &quot; können Sie Informationen zur Eigenschaft&quot; Eigenschaften&quot; eines bestimmten Benutzers abrufen.

## S-T {#s-t}

**Segment**

Ein Segment (oder eine Zielgruppe) ist eine Gruppe von Benutzern, die gemeinsame Attribute verwenden.

Siehe [Segmente: Zweck, Zusammensetzung und Regeln](../features/segments/segments-purpose.md).

<br> 

**Segment Addressable Audience**

In [Addressable Audience](/help/using/features/addressable-audiences.md)stellt diese Metrik die Anzahl der Benutzer dar, die während des Berichtrückblickzeitraums zu dem Segment gehören und auf Ihrer Site eine aktive ID-Synchronisierung haben. Segmente können Ihre eigenen Erstanbieterdaten sowie Drittanbieter- und Drittanbieter-Daten über Eigenschaften enthalten, die im [Audience Marketplace erworben](/help/using/features/audience-marketplace/audience-marketplace.md)wurden.

<br> 

**Gesamtanzahl der Segmente**

In [Addressable Audience](/help/using/features/addressable-audiences.md)stellt diese Metrik eine Anzahl aller Geräte dar, die während des Berichtrückblickzeitraums ein Mitglied Ihres Segments waren.

<br> 

**Segmentübereinstimmungsrate**

Segment Addressable Audience ÷ Gesamtsegmentpopulation ausgedrückt als %. Siehe [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**Signal**

Signale sind die kleinsten Dateneinheiten in [!DNL Audience Manager] und werden als Schlüssel-Wert-Paare ausgedrückt.

Siehe [Signale, Eigenschaften und Segmente](../reference/signal-trait-segment.md).

<br> 

**Trait-**

Eine Eigenschaft ist eine Kombination aus einem oder mehreren Signalen. Siehe [Signale, Eigenschaften und Segmente](../reference/signal-trait-segment.md).

<br> 

**Trait-Population**

Siehe [Eigenschaften von Eigenschaften und Segmentpopulationen im Segmentaufbau](../features/segments/segment-builder-data.md).

**TTL (Time-to-Live)**

TTL definiert, wie viele Tage ein qualifizierter Besucher in einer Eigenschaft verbleibt. TTL wird auf Eigenschaften und nicht auf Segmenten festgelegt. Besucher verlassen sich aus einem Segment, wenn vor dem Ende des TTL-Intervalls keine qualifizierende Eigenschaft angezeigt wird. Read more in [Segment and Trait Time-to-Live Explained](/help/using/features/traits/segment-ttl-explained.md).

<br> 

## U-V {#u-v}

**UUID**

[!DNL Audience Manager] Unique User ID. Siehe [Index von IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Visitor ID**

Der [!DNL Experience Cloud] ID-Dienst (ehemals Besucher-ID) stellt eine universelle, beständige ID bereit, die Ihre Besucher für alle Lösungen in der Regel [!DNL Experience Cloud]identifiziert.

Weitere Informationen finden Sie in der Dokumentation [zum Experience Cloud ID-Dienst](https://marketing.adobe.com/resources/help/en_US/reference/marketing-cloud-id-service.html) .

## W-X-Y-Z {#w-z}
