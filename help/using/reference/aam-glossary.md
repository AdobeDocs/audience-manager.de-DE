# Glossar{#glossary}

Definitionen und Links zur weiteren Lesbarkeit.

## A-B {#a-b}

**Algorithmische Modellierung**

Use [!UICONTROL Algorithmic Modeling] as a means of extending reach beyond the core of users you've identified. Mit dieser Funktion können Sie neue, einzigartige Zielgruppen durch automatisierte Datenanalyse entdecken. Manage your [!UICONTROL Algorithmic Models] in **[!UICONTROL Audience Data > Models]**.

See [Understanding Algorithmic Models](../features/algorithmic-models/understanding-models.md#understanding-models).

<br> 

**BAAAM**

[!UICONTROL Bulk Management Tools]. The [!UICONTROL Bulk Management Tools] in [!DNL Audience Manager] are a Microsoft Excel-based set of tools that let you create, modify or delete multiple objects at once with a single operation. Sie können mit Datenquellen, abgeleiteten Signalen, Zielen, Ordnern, Segmenten und Eigenschaften arbeiten. The feature uses a Microsoft Excel spreadsheet with macros that make secure, authenticated calls to the [!DNL Audience Manager] APIs.

See [Bulk Management Tools](../reference/bulk-management-tools/bulk-management-intro.md).

## C-D {#c-d}

**CDF**

[!UICONTROL Customer Data Feed]. A [!UICONTROL CDF] file represents a bulk download of data collected by [!DNL Audience Manager] and enables you to work with [!DNL Audience Manager] data outside of the limits imposed by our user interface. A [!UICONTROL CDF] file contains the same data that an [!DNL Audience Manager] event call ( `/event`) sends to our servers. Dazu gehören Daten wie Benutzer-IDs, Eigenschaften-IDs, Segment-IDs und alle anderen Parameter, die durch einen Ereignisaufruf erfasst werden.

See [Customer Data Feeds](../features/cdf-files.md).

<br> 

**CRM-ID**

Die CRM-ID ist die ID, mit der Kunden Benutzer in ihrem eigenen CRM-System identifizieren. Anstelle der CRM-ID verwenden wir den Begriff DPUUID in Audience Manager.

See DPUUID in the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Kundenadressierbare Zielgruppe**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents devices that:
* Entweder eine regelbasierte oder eine aufgenommene Eigenschaft während des Look-Back-Fensters
   **UND**
* Eine ID wird unabhängig vom Synchronisierungszeitpunkt mit dem ausgewählten Ziel synchronisiert.

<br> 

**Kundenattribute**

See [Customer Attributes](https://marketing.adobe.com/resources/help/en_US/mcloud/attributes.html) in the [!DNL Experience Cloud Core Services] product documentation.

<br> 

**Kundenübereinstimmungsrate**

Customer Addressable Audience ÷ Customer Total Audience Express as a %. See [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**Zielgruppe für Kunden insgesamt**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents a count of devices that have realized either a rule-based trait on your properties or an onboarded trait from your offline files during the look-back window.

<br> 

**demdex.net**

Demdex.net is a legacy domain controlled by [!DNL Adobe]. It reflects [!DNL Audience Manager]'s original, pre-acquisition name ( [!DNL Demdex]). [!DNL Adobe] im [!DNL Demdex] Jahr 2011 erworben und das Unternehmen erneut [!DNL Audience Manager]markiert. All HTTP calls to `demdex.net` domains are calls sent in to [!DNL Adobe].

Siehe [Aufrufe an die Domäne „demdex.net“](../reference/demdex-calls.md).

<br> 

**DAID**

[!UICONTROL Device Advertising IDs] sind eindeutige Gerätekennungen zur Identifizierung eines Mobilgeräts. Diese IDs werden vom Gerätehersteller und nicht von Adobe zugewiesen. We support iOS and Android device IDs in [!DNL Audience Manager].

See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Ziel**

In [!DNL Audience Manager], a destination is any other system (ad server, DSP, ad network, etc.) , für die Sie Daten freigeben möchten. The [!UICONTROL Destination Builder] in our UI provides the tools that let you create and manage these data delivery processes. [!DNL Audience Manager] Zielfunktionen befinden **[!UICONTROL Audience Data > Destinations]** sich in.

<br> 

**DIL**

The [!UICONTROL Data Integration Library] is an API library used by [!DNL Audience Manager] to collect user interaction data. See [Data Integration Library (DIL) API](../dil/dil-overview.md).

<br> 

**dpm**

[!UICONTROL Data Provider Match]. It tells internal [!DNL Adobe] systems that a call from [!DNL Audience Manager] or the ID service is passing in customer data for synchronization or requesting an ID. Siehe [Aufrufe an die Domäne „demdex.net“](../reference/demdex-calls.md).

## E-F {#e-f}

**Experience Cloud ID (ECID)**

Previously named the [!DNL Marketing Cloud] ID (MID or MCID). The [!DNL Experience Cloud] ID is central to the ID Service. Es handelt sich um einen eindeutigen und beständigen Bezeichner für Ihre Site-Besucher. See Cookies and the [Experience Cloud ID Service](https://marketing.adobe.com/resources/help/en_US/mcvid/mcvid_cookies.html).

<br> 

**Ordnereigenschaften**

Automatische Gruppierung von Eigenschaften in Ihrer Ordnertaxonomie. Jeder Ordner in Ihrer Hierarchie erstellt automatisch eine Eigenschaft, die zur Definition von Segmenten verwendet werden kann.

See [Folder Traits: About](../features/traits/about-folder-traits.md).

<br> 

**Häufigkeitsklick**

Ein Limit, mit dem ein Advertiser einem Endbenutzer eine bestimmte kreative Darstellung anzeigen möchte. You can configure various frequency capping expressions in [!UICONTROL Segment Builder].

See [Recency and Frequency](../features/segments/recency-and-frequency.md).

## G-H {#g-h}

**GAID**

Google Advertising ID, die eindeutige Geräte-ID, die Google auf Hardware-Geräten zuordnet, die das Android-Betriebssystem ausführen. See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**GUID**

Ein Akronym für Global eindeutige Kennung. We don't use the term GUID in [!DNL Audience Manager]. In our case, the GUID is the [!DNL Audience Manager] UUID.
See [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

## I-J {#i-j}

**IDFA**

ID für Advertiser, die eindeutige Geräte-ID, die Apple seinen Produkten zuordnet. See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Inbound**

The process by which you can send audience data from other sources to [!DNL Audience Manager]. See [Sending Audience Data](/help/using/integration/sending-audience-data/send-audience-data.md).

<br> 

**Integrationscode**

When working with the [!DNL Audience Manager] UI or API, you have the option of adding an integration code when creating traits, segments, or data sources. Integrationscodes dienen in diesen Fällen unterschiedlichen Zwecken:

* [!UICONTROL Traits]: Ein Integrationscode ist ein Feld für eine ID, SKU oder einen anderen Wert, der von Ihren internen Geschäftsprozessen verwendet wird. Optional.
* [!UICONTROL Segments]: Ein Integrationscode ist ein Feld für eine benutzerdefinierte ID oder andere unternehmensspezifische Informationen. Optional.
* [!UICONTROL Data Sources]: Integrationscodes sind erforderlich, wenn Sie geräteübergreifende Datenquellen erstellen, den Experience Cloud ID-Dienst verwenden oder mit [!UICONTROL Profile Merge Rules]arbeiten möchten. See [Create a Data Source](../features/manage-datasources.md#create-data-source) for more information.

## K-L {#k-l}

**[!UICONTROL Look-alike modeling]**

See [Algorithmic Modeling](../reference/aam-glossary.md#a-b).

## M-N {#m-n}

**MCID**, **MID**

See the [Experience Cloud ID](../reference/aam-glossary.md#e-f).

## O-P {#o-p}

**PCS**

[!UICONTROL Profile Cache Server]. The [!UICONTROL PCS] is a large database, running on Apache Cassandra. It stores data received for active users from server-to-server transfers and the [!UICONTROL DCS]. [!UICONTROL PCS] Daten bestehen aus Geräte-IDs, authentifizierten Profil-IDs und den zugehörigen Eigenschaften.

See [Data Collection Components](../reference/system-components/components-data-collection.md).

<br> 

**Profillink**

See [Profile Merge Rule Options Defined](../features/profile-merge-rules/merge-rule-definitions.md).

<br> 

**Regeln für die Profilzusammenführung**

[!UICONTROL Profile Merge Rules] können Sie steuern, welche Datenart für die Segmentierung [!DNL Audience Manager] verwendet werden soll.

See [Profile Merge Rule Options Defined](../features/profile-merge-rules/merge-rule-definitions.md).

## Q-R {#q-r}

**Neuerstellung**

Die Aktion, mit der sich ein Besucher auf Ihrer Site für eine Eigenschaft qualifiziert. You can use the [Visitor Profile Viewer](../features/visitor-profile-viewer.md) tool to obtain information on trait realization by a specific user.

## S-T {#s-t}

**Segment**

Ein Segment (oder eine Zielgruppe) ist eine Gruppe von Benutzern, die gemeinsame Attribute verwenden.

See [Segments: Purpose, Composition, and Rules](../features/segments/segments-purpose.md).

<br> 

**Segment Addressable Audience**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents the number of users who have belonged to the segment during the report look-back period and have an active ID sync on your site. Segments can include your own first-party data and second party and third party data, via traits acquired in the [Audience Marketplace](/help/using/features/audience-marketplace/audience-marketplace.md).

<br> 

**Gesamtanzahl der Segmente**

In [Addressable Audience](/help/using/features/addressable-audiences.md), this metric represents a count of all the devices that were a member of your segment during the report look-back period.

<br> 

**Segmentübereinstimmungsrate**

Segment Addressable Audience ÷ Gesamtsegmentpopulation ausgedrückt als %. See [Addressable Audience](/help/using/features/addressable-audiences.md).

<br> 

**Signal**

Signals are the smallest data units in [!DNL Audience Manager] and are expressed as key-value pairs.

See [Signals, Traits, and Segments](../reference/signal-trait-segment.md).

<br> 

**Trait-**

Eine Eigenschaft ist eine Kombination aus einem oder mehreren Signalen. See [Signals, Traits, and Segments](../reference/signal-trait-segment.md).

<br> 

**Trait-Population**

See [Trait and Segment Population Data in Segment Builder](../features/segments/segment-builder-data.md).

**TTL (Time-to-Live)**

TTL definiert, wie viele Tage ein qualifizierter Besucher in einer Eigenschaft verbleibt. TTL wird auf Eigenschaften und nicht auf Segmenten festgelegt. Besucher verlassen sich aus einem Segment, wenn vor dem Ende des TTL-Intervalls keine qualifizierende Eigenschaft angezeigt wird. Read more in [Segment and Trait Time-to-Live Explained](/help/using/features/traits/segment-ttl-explained.md).

<br> 

## U-V {#u-v}

**UUID**

[!DNL Audience Manager] Unique User ID. See the [Index of IDs in Audience Manager](../reference/ids-in-aam.md).

<br> 

**Visitor ID**

The [!DNL Experience Cloud] ID Service (formerly visitor ID) provides a universal, persistent ID that identifies your visitors across all the solutions in the [!DNL Experience Cloud].

See the [Experience Cloud ID Service](https://marketing.adobe.com/resources/help/en_US/reference/marketing-cloud-id-service.html) documentation.
